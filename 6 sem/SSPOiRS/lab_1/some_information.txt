На клиенте требуется настроить IP, по которому будет подключен сам клиент. 

В этой программе клиент может подключаться к серверу неограниченное количество раз (пока его ответ == "да"). Но это происходит лишь в тот момент, пока связь с сервером оборвана.

При подключении клиента к серверу на серверной консоли выводятся адрес клиента - ip и порт.

•	Локальный порт: Это порт на вашем компьютере, который используется для инициирования соединения.
•	Удаленный порт: Это порт на удаленном компьютере (сервере), к которому вы пытаетесь подключиться.

8080 – удаленка (используется на сервере и на клиенте)
Ххххх – локалка (возникает у клиента при подключении)

Так же, есть одна заминка. Здесь нельзя было использовать метод sendall(). Вместо его требовалось использовать send().

int recv( SOCKET s, char* buf, int len, int flags );
int send( SOCKET s, const char* buf, int len, int flags );

Функция send(), по аналогии с функциями записи в файл write(), посылает данные из буфера (в количестве значения параметра len) в сокет (то есть – в сеть), а recv() – получает из сети указанное количество байт и помещает их в буфер. Последние параметры этих функций используются как настройки пересылки; можно использовать значение 0. Функция recv() является так же
блокирующей. 

Так же тут как бы реализована попытка восстановить передачу данных при разрыве соединения, но по факту - нет.
